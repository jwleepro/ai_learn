# 00. 딥러닝 기초를 “코드로” 잡기 (LLM 들어가기 전)

목표: 수학을 외우기보다, **LLM을 구성하는 계산**(선형결합/행렬곱/경사하강/활성화)을 코드로 체감합니다.

> 이 레슨은 Week 0의 “기초 다지기” 파트입니다.  
> 끝까지 이해 못해도 괜찮고, **직접 실행해보고 결과가 왜 그런지**만 잡으면 됩니다.

---

## 1) 뉴런 1개는 중학교 수학이다: `y = w*x + b`

- `x`: 입력
- `w`: 가중치(기울기 느낌)
- `b`: 편향(기본값 느낌)

코드로는 그냥:

```python
def simple_neuron(x, w, b):
    return (w * x) + b
```

LLM도 결국 이런 계산이 엄청 많이 쌓여 있습니다.

---

## 2) 행렬 곱셈 = 반복문을 한 번에 처리

입력이 여러 개이고 출력도 여러 개면, `for`를 돌릴 수도 있지만 딥러닝은 보통 **행렬곱으로 한 방에** 계산합니다.

예: `[햄버거, 감자튀김, 콜라]` 판매량으로 `[매출, 순이익]`을 계산:

- 입력 `x` (3개)
- 가중치 `W` (2x3)
- 편향 `b` (2개)
- 출력 `y = W @ x + b`

실습 코드는 `llm_from_scratch/code/week0_dl_basics.py`에 있습니다.

## 3) 학습(Training) = `w, b`를 자동으로 찾는 과정

모델이 어떤 정답 `y_true`를 맞히게 하려면,
모델 예측 `y_pred`가 정답과 가까워지도록 `w, b`를 조정해야 합니다.

가장 흔한 방식이 **경사하강법(Gradient Descent)** 입니다:

- “지금 손실(loss)을 줄이는 방향”으로 조금씩 이동

비유: 눈을 감고 산에서 내려온다고 상상하세요.
발밑을 더듬어 “어느 쪽이 가장 가파른 내리막인지” 알아내고, 그 방향으로 한 걸음 옮깁니다.
이것을 반복하면 결국 낮은 곳(loss가 작은 곳)에 도달합니다.

> **기울기(gradient)**: “한 걸음 옮길 방향”을 알려주는 값
>
> **학습률(lr)**: “한 걸음의 크기” — 너무 크면 넘어가고, 너무 작으면 느림

우리는 아주 단순한 **직선 맞추기(선형회귀)**로 감을 잡습니다:

- 데이터 `(x, y)`가 있을 때 `y ≈ w*x + b`
- MSE loss(평균제곱오차: 예측과 정답의 차이를 제곱해서 평균낸 것)를 줄이도록 `w, b`를 업데이트

### 왜 저 계산식이 나올까? (편미분 없이 이해하기)

MSE loss는 "오차의 제곱 평균"입니다:

- `loss = mean( (y_pred - y)² )`
- 여기서 `y_pred = w*x + b`

**w를 살짝 바꾸면 loss가 얼마나 변할까?**

숫자 예시로 확인해봅시다. 데이터가 1개(`x=3, y=7`)이고 지금 `w=2, b=0`이라면:

| | w=2 | w=2.01 (살짝 증가) |
|---|---|---|
| y_pred = w*x + b | 2×3 = 6 | 2.01×3 = 6.03 |
| err = y_pred - y | 6 - 7 = **-1** | 6.03 - 7 = -0.97 |
| loss = err² | 1 | 0.9409 |

w를 0.01 올렸더니 loss가 `1 → 0.9409`, 즉 **0.0591 줄었습니다**.

이번엔 `x=100`인 데이터로 같은 실험을 해보면:

| | w=2 | w=2.01 |
|---|---|---|
| y_pred | 200 | 201 |
| loss 변화 | | 훨씬 크게 변함 |

`x`가 클수록 w를 살짝 바꿨을 때 loss가 크게 흔들립니다.
정리하면, **w를 바꿨을 때 loss가 얼마나 변하는지**(= 기울기)는 두 가지에 비례합니다:

1. **오차(err)** — 많이 틀릴수록 변화가 큼
2. **입력(x)** — 입력이 클수록 변화가 큼

그래서 w의 기울기 = `err * x`, 데이터가 여러 개니까 평균: **`mean(err * x)`**

**b는?**

`y_pred = w*x + b`에서 `b`를 1 올리면, `x`가 뭐든 `y_pred`는 무조건 1 올라갑니다.
그래서 기울기는 오차만 남습니다: **`mean(err)`**

코드로 보면:

```python
dw = (2/n) * sum(err * x)   # w를 얼마나 바꿀지
db = (2/n) * sum(err)        # b를 얼마나 바꿀지
w -= lr * dw                  # 기울기 반대 방향으로 이동
b -= lr * db
```

> `2`가 곱해진 이유: `x²`에서 `x`가 1 늘면 변화량에 `2`가 붙는 규칙이 있습니다(고등 수학에서 다루는 내용이므로 지금은 "그런 규칙이 있다"만 알면 충분합니다).
> 어차피 상수이므로 학습률(`lr`)에 흡수시켜도 결과는 같습니다.
> 더 깊이 알고 싶으면 "MSE gradient"를 검색해보세요 — 지금은 **"오차 × 입력 = w 기울기"**만 기억하면 됩니다.

실습 스크립트:

```powershell
python llm_from_scratch/code/demo_week0_dl_basics.py
```

## 4) 활성화 함수(Activation): “비선형성” 넣기

선형만(행렬곱만) 계속 쌓으면, 결국 “큰 선형”이 됩니다.  
그래서 중간에 비선형 함수(예: ReLU)를 넣어 더 똑똑한 함수가 되게 합니다.

- ReLU: `max(0, x)`
- tanh, sigmoid 등(지금은 이름만)

LLM의 MLP(FFN) 안에도 이런 비선형이 들어갑니다.
