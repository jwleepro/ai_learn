# Core 용어집(Weeks 0~7)

이 파일은 **Core 트랙을 먼저 따라가는 사람**을 위한 짧은 용어집입니다.

- 먼저 볼 대상: Week 0~7
- 나중에 볼 대상: 파인튜닝/LoRA/운영 용어는 [GLOSSARY.md](GLOSSARY.md)

> 팁: 수식이 보이면 값보다 **shape(배열 크기)** 를 먼저 보세요.
> 예: `(B, T, D)` = 배치 B개, 토큰 T개, 각 토큰 벡터 길이 D

---

## 토큰/데이터

- **token(토큰)**: 텍스트를 자른 최소 단위. 이 프로젝트는 먼저 글자 토큰으로 시작합니다.
- **vocab(어휘)**: 토큰 목록. 글자 토큰이면 "등장한 글자 목록"입니다.
- **token id**: 각 토큰에 붙인 정수 번호.
- **encode / decode**
  - `encode(text) -> list[int]`: 텍스트를 id로 바꾸기
  - `decode(ids) -> str`: id를 다시 텍스트로 바꾸기
- **corpus(코퍼스)**: 학습에 쓰는 텍스트 덩어리.
- **context(컨텍스트)**: 다음 토큰을 맞힐 때 참고하는 이전 토큰들.

---

## 확률/출력

- **P(A | B)**: "B일 때 A가 일어날 확률"
  - 예: `P(next | prev)` = 이전 글자가 `prev`일 때 다음 글자가 `next`일 확률
- **logits(로짓)**: softmax 직전의 점수들.
- **softmax**: 점수를 확률분포로 바꾸는 함수.
  - 출력값은 0~1 사이
  - 전체 합은 1
- **확률분포**: 각 토큰이 나올 확률을 모아 둔 벡터.
- **sampling(샘플링)**: 확률분포에서 실제 토큰을 하나 뽑는 과정.

---

## 학습

- **loss(손실)**: 모델이 정답을 얼마나 못 맞혔는지 숫자 1개로 만든 것. 작을수록 좋습니다.
- **cross-entropy**: 다음 토큰 맞히기에서 자주 쓰는 loss.
  - 정답 토큰에 높은 확률을 주면 loss가 내려갑니다.
- **perplexity(ppl)**: loss를 사람이 보기 쉬운 값으로 바꾼 것. 작을수록 좋습니다.
- **gradient(기울기)**: 파라미터를 어느 방향으로 바꿔야 loss가 줄어드는지 알려주는 값.
- **gradient descent(경사하강법)**: 기울기 반대 방향으로 조금씩 이동하는 업데이트 방법.
  - 핵심 한 줄: `파라미터 = 파라미터 - 학습률 * 기울기`
- **learning rate(lr)**: 한 번 업데이트할 때 얼마나 크게 움직일지.
- **epoch**: 학습 데이터를 한 바퀴 돈 것.
- **batch**: 한 번에 묶어서 처리하는 샘플들.
- **train / val split**: 학습용과 검증용 데이터를 나누는 것.
- **overfitting(과적합)**: train에서는 좋아지는데 val에서는 나빠지는 현상.

---

## 모델 구성 요소

- **embedding(임베딩)**: 토큰 id를 벡터로 바꾸는 테이블.
  - 토큰 id가 `i`면 보통 `E[i]`를 꺼냅니다.
- **MLP**: 선형 계산 + 비선형 함수(ReLU, tanh 등)를 쌓은 작은 신경망.
- **FFN**: Transformer 안에 들어 있는 MLP 부분.
- **Residual(스킵 연결)**: `x = x + f(x)`처럼 원래 값을 더해주는 연결.
- **LayerNorm**: 벡터 값들의 평균과 퍼진 정도를 맞춰서 학습을 안정화하는 계산.

---

## Attention / Transformer

- **self-attention**: 현재 위치가 과거의 어느 위치를 얼마나 볼지 정하는 계산.
- **dot product(내적)**: 같은 위치끼리 곱해서 전부 더한 값.
  - attention에서는 두 토큰이 얼마나 관련 있는지 점수로 쓰입니다.
- **causal mask**: 미래 토큰을 못 보게 가리는 장치.
- **head(헤드)**: attention을 여러 관점으로 나눠 보는 단위.
- **d_model**: 토큰 벡터 길이.
- **d_head**: 한 head가 쓰는 벡터 길이.

---

## 생성 옵션

- **temperature**: 랜덤성을 조절하는 값.
  - `< 1`: 더 보수적
  - `= 1`: 기본
  - `> 1`: 더 랜덤
- **top-k**: 확률이 큰 상위 k개만 남기는 방법.
- **top-p**: 확률이 큰 것부터 더해서 누적합이 p가 될 때까지 남기는 방법.
- **smoothing(Add-k)**: 카운트 모델에서 안 본 전이도 0이 되지 않게 조금 확률을 주는 방법.

---

## shape 표기

- `V`: vocab 크기
- `T`: 시퀀스 길이
- `D`: 임베딩/모델 차원
- `H`: hidden 차원
- `B`: batch 크기
- 예: `(B, T, D)` = 배치 B개, 토큰 T개, 각 토큰 벡터 길이 D

---

## 코드에서 만나는 수학 함수

공식을 외울 필요는 없습니다. 역할만 알면 충분합니다.

- **exp**: 점수를 양수 값으로 바꾸는 함수. softmax 안에서 씁니다.
- **log**: exp의 반대 방향 함수. cross-entropy에서 씁니다.
- **sqrt**: 제곱근. attention 점수가 너무 커지지 않게 나눌 때 씁니다.
- **분산(variance)**: 값들이 평균에서 얼마나 퍼져 있는지 나타내는 수. LayerNorm에서 씁니다.
